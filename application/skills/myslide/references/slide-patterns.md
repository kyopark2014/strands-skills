# Slide Layout Patterns

Each pattern below includes PptxGenJS code snippets using the AWS_THEME constants
from aws-theme.md. Adapt content but preserve the visual structure.

Background images are generated by `scripts/create_aws_slide.py backgrounds`.

## Content Card

The most common layout: rounded container with icon + vertical separator + text.
Ideal for explaining a single concept with 2-4 bullet points.

**Icon selection guidelines:**
- Use **AWS service icons** (from `icons/` directory) when the content relates to an AWS service
- Use **native PPTX shapes** (accent bars, colored dots, numbered shapes) instead of react-icons
  when the meaning is simple (e.g., agenda lists, feature highlights)
- **Avoid react-icons** that render as complex multi-element SVGs (e.g., FaListOl renders as
  "1,2,3" with lines, which looks messy at small sizes). If you must use react-icons,
  prefer simple single-shape icons (FaRocket, FaShieldAlt, FaCog)
- For **Agenda slides**, use numbered text (01, 02...) with orange accent bars instead of
  list icons -- the numbers themselves serve as the visual element

```javascript
let slide = pres.addSlide();
slide.background = { data: bgContentBase64 };

// Title
slide.addText("Data privacy and localization", {
  ...AWS_THEME.positions.title,
  fontSize: 36, fontFace: AWS_THEME.fonts.heading,
  color: AWS_THEME.colors.white, bold: true
});

// Content card container (rounded rectangle with pink border)
slide.addShape(pres.shapes.ROUNDED_RECTANGLE, {
  x: 0.42, y: 1.41, w: 12.5, h: 4.84,
  fill: { color: AWS_THEME.colors.bgBase, transparency: 60 },
  line: { color: AWS_THEME.colors.magenta, width: 1 },
  rectRadius: 0.15
});

// Icon area (embed SVG-rendered PNG)
slide.addImage({
  data: iconBase64,  // Pre-rendered icon PNG
  x: 1.2, y: 2.8, w: 1.4, h: 1.4
});

// Vertical separator line
```

## Architecture Diagram

**IMPORTANT: Prefer native PPTX elements over embedded images.**
Build diagrams using PptxGenJS shapes (ROUNDED_RECTANGLE, OVAL, LINE), text,
and AWS service icon PNGs. Only use SVG-to-PNG embedding as a last resort
(e.g., for extremely complex diagrams that cannot be composed from shapes).
When embedding images, always preserve the original aspect ratio.

**CRITICAL: Every service box in an architecture diagram MUST include its AWS
service icon.** Read the icon SVG from the skill's `icons/` directory, convert
to PNG via sharp, and embed it inside each service box. Architecture diagrams
without service icons look empty and unprofessional. The icon should be placed
at the top or left of each service box (0.45"x0.45" minimum size).

```javascript
// ALWAYS load and embed service icons for every service box
const iconSvg = fs.readFileSync(path.join(SKILL_DIR, 'icons', 'bedrock.svg'), 'utf-8');
const iconPng = await sharp(Buffer.from(iconSvg)).resize(256, 256).png().toBuffer();
const iconBase64 = 'image/png;base64,' + iconPng.toString('base64');

// Then use iconBase64 in addImage() inside each service box
```

### Arrow Routing Rules (CRITICAL for Architecture Diagrams)

**Arrows must NEVER pass through another service box.** This is the most common
architecture diagram defect. Before drawing any arrow, check if there is a service
box between the source and target. If so, use ONE of these strategies:

1. **Route around**: Use two LINE segments (horizontal + vertical) to go around the
   blocking box. PptxGenJS lines can only be straight, so use two segments to make
   an L-shaped or Z-shaped route.
2. **Reposition boxes**: Move the blocking box to a different row/column so the
   arrow has a clear path.
3. **Connect via intermediate**: Route source → blocking box → target if there's
   a logical flow through it.

```javascript
// WRONG: Direct diagonal arrow that crosses through another box
addArrow(slide, { x1: 5, y1: 3, x2: 10, y2: 5 }); // may cross boxes in between

// CORRECT: L-shaped route around blocking boxes
// Horizontal segment first
slide.addShape(pres.shapes.LINE, {
  x: 5, y: 3, w: 3, h: 0,
  line: { color: AWS_THEME.colors.slateGray, width: 1 }
});
// Vertical segment with arrow
slide.addShape(pres.shapes.LINE, {
  x: 8, y: 3, w: 2, h: 2,
  line: { color: AWS_THEME.colors.slateGray, width: 1, endArrowType: "triangle" }
});
```

**Validation step**: After placing all boxes and arrows, mentally trace each arrow
path and verify it does not cross any box's bounding rectangle. If it does, reroute.

### Native Architecture Pattern (Preferred)

```javascript
let slide = pres.addSlide();
slide.background = { data: bgContentBase64 };

// Title
slide.addText("Amazon Bedrock API access flows", {
  ...AWS_THEME.positions.title,
  fontSize: 36, fontFace: AWS_THEME.fonts.heading,
  color: AWS_THEME.colors.white, bold: true
});

// Service box example (repeat for each service)
function addServiceBox(slide, { x, y, w, h, name, desc, iconData, borderColor }) {
  slide.addShape(pres.shapes.ROUNDED_RECTANGLE, {
    x, y, w, h,
    fill: { color: AWS_THEME.colors.darkNavy },
    line: { color: borderColor || AWS_THEME.colors.slateGray, width: 1 },
    rectRadius: 0.1
  });
  if (iconData) {
    slide.addImage({ data: iconData, x: x + 0.2, y: y + 0.15, w: 0.45, h: 0.45 });
  }
  slide.addText(name, {
    x: x + (iconData ? 0.75 : 0.2), y: y + 0.15, w: w - 1.0, h: 0.4,
    fontSize: 15, fontFace: AWS_THEME.fonts.heading,
    color: AWS_THEME.colors.white, bold: true
  });
  if (desc) {
    slide.addText(desc, {
      x: x + 0.2, y: y + 0.65, w: w - 0.4, h: 0.6,
      fontSize: 10, fontFace: AWS_THEME.fonts.body,
      color: AWS_THEME.colors.slateGray, valign: "top"
    });
  }
}

// Arrow connector — ALWAYS verify the path is clear of other boxes before drawing
function addArrow(slide, { x1, y1, x2, y2, color, label }) {
  slide.addShape(pres.shapes.LINE, {
    x: x1, y: y1, w: x2 - x1, h: y2 - y1,
    line: { color: color || AWS_THEME.colors.slateGray, width: 1, endArrowType: "triangle" }
  });
  if (label) {
    slide.addText(label, {
      x: (x1 + x2) / 2 - 0.5, y: (y1 + y2) / 2 - 0.2, w: 1.0, h: 0.3,
      fontSize: 10, fontFace: AWS_THEME.fonts.body,
      color: AWS_THEME.colors.slateGray, align: "center"
    });
  }
}

// Layer label (vertical text on left)
function addLayerLabel(slide, { x, y, text, color }) {
  slide.addText(text, {
    x, y, w: 0.5, h: 1.5,
    fontSize: 11, fontFace: AWS_THEME.fonts.heading,
    color: color || AWS_THEME.colors.magenta, bold: true,
    rotate: 270, align: "center"
  });
}
```

### Image Embedding Fallback (Last Resort)

When embedding pre-rendered images, **always preserve aspect ratio**:

```javascript
// Calculate dimensions preserving aspect ratio
const origW = 2400, origH = 1200;
const maxW = 12.5, maxH = 5.5;
const scale = Math.min(maxW / origW, maxH / origH);
const dispW = origW * scale;
const dispH = origH * scale;
const centerX = (13.33 - dispW) / 2;

slide.addImage({
  data: archDiagramBase64,
  x: centerX, y: 1.2, w: dispW, h: dispH
});
```

**SVG Generation (fallback only):**
Use `scripts/create_aws_slide.py svg-diagram` or manually craft SVG with:
- Nested rectangles for account/VPC/subnet containers
- Color-coded borders: customer=white, AWS-owned=C91F8A, data-plane=010135
- Arrow connectors between components
- AWS service icon placeholders (simple geometric shapes with labels)

## Multi-Card Grid

Multiple semi-transparent rounded cards arranged in a grid. The dominant pattern
in AWS 2025 presentations (used in ~49% of slides). Each card contains a title,
description, and optional icon.

```javascript

  ...AWS_THEME.positions.title,
  fontSize: 36, fontFace: AWS_THEME.fonts.heading,

});

// Cards: 2 columns x 2 rows
const cards = [
  { title: "Orchestration", desc: "Coordinate multi-step workflows", icon: "step-functions" },
  { title: "Knowledge Base", desc: "RAG with managed vector store", icon: "bedrock" },
  { title: "Guardrails", desc: "Content filtering and safety", icon: "shield" },
  { title: "Code Interpreter", desc: "Execute code in sandbox", icon: "lambda" },
];

const cardW = 5.8;
const cardH = 2.2;
const gap = 0.4;
const startX = 0.6;
const startY = 1.3;

cards.forEach((card, i) => {
  const col = i % 2;
```

## Gradient Border Cards

Light-colored cards with gradient-colored borders on a dark background.
Each card has a distinct border color from the AWS palette, creating a
visually striking but clean comparison layout. Based on the AWS 2025
"Built for builders" slide pattern.

**Key design characteristics:**
- **Light card fills** (`F2F4F4` or `FFFFFF`) on dark slide background — high contrast
- **Gradient-colored borders** — each card has a different accent color
- **Bold card title** at top, check-mark bullet list below
- 3-4 cards in a single row (responsive to content amount)

Since PptxGenJS doesn't support gradient borders natively, use solid
accent colors from the 5-color palette. For true gradient borders,
render SVG cards and embed as PNG images.

### Simple Approach: Solid Accent Borders (Recommended)

```javascript

  {
```

### Advanced: SVG Gradient Border Cards

For true gradient borders (e.g., orange→red fade), render each card frame
as an SVG and embed as a PNG image, then overlay text on top:

```javascript
async function createGradientBorderCard(w, h, colors, direction = 'to bottom') {
  const pxW = Math.round(w * 96);
  const pxH = Math.round(h * 96);
  const [c1, c2] = colors;
  const borderW = 3;  // border width in pixels
  const radius = 12;

  const svg = `<svg xmlns="http://www.w3.org/2000/svg" width="${pxW}" height="${pxH}">
    <defs>
      <linearGradient id="border" x1="0" y1="0" x2="0" y2="1">
        <stop offset="0%" stop-color="#${c1}"/>
        <stop offset="100%" stop-color="#${c2}"/>
      </linearGradient>
    </defs>
    <rect x="0" y="0" width="${pxW}" height="${pxH}" rx="${radius}"
          fill="url(#border)"/>
    <rect x="${borderW}" y="${borderW}"
          width="${pxW - borderW * 2}" height="${pxH - borderW * 2}"
          rx="${radius - 1}" fill="#F2F4F4"/>
  </svg>`;

  const png = await sharp(Buffer.from(svg)).png().toBuffer();
  return 'image/png;base64,' + png.toString('base64');
}

// Usage: each card gets a different gradient border
const cardFrames = await Promise.all([
  createGradientBorderCard(2.9, 4.2, ['F66C02', 'C91F8A']),  // Orange→Magenta
  createGradientBorderCard(2.9, 4.2, ['5600C2', 'C91F8A']),  // Purple→Magenta
  createGradientBorderCard(2.9, 4.2, ['00A0C8', '5600C2']),  // Teal→Purple
  createGradientBorderCard(2.9, 4.2, ['2D7CFB', '5600C2']),  // Blue→Purple
]);

// Embed card frame as image, then add text on top
cardFrames.forEach((frame, i) => {
  const x = startX + i * (cardW + gap);
  slide.addImage({ data: frame, x: x, y: cardY, w: cardW, h: cardH });
  // Add title and bullet text on top (same as simple approach above)
});
```

**When to use which approach:**
- **Solid borders (simple)**: Fast, reliable, sufficient for most presentations
- **SVG gradient borders (advanced)**: When matching a specific design reference that
  uses gradient borders (like the AWS 2025 "Built for builders" style)

## Comparison Table

Multi-row, multi-column comparison table with header row and alternating fills.
Ideal for option comparisons, feature matrices, and specification tables.

```javascript
let slide = pres.addSlide();
slide.background = { data: bgContentBase64 };

slide.addText("Option A vs Option B", {
  ...AWS_THEME.positions.title,
  fontSize: 36, fontFace: AWS_THEME.fonts.heading,
  color: AWS_THEME.colors.white, bold: true
});

const headers = ["Criteria", "Option A: Amazon Quick", "Option B: Custom Chatbot"];
const rows = [
  ["Deployment Speed", "Fast (days)", "Medium (weeks~months)"],
  ["Customization", "Limited (managed)", "High (custom UI/UX)"],
  ["Cost Model", "Per-user subscription", "Pay-per-use (token-based)"],
  ["Operations", "Minimal (AWS managed)", "Medium (self-operated)"],
  ["Model Selection", "Quick's built-in model", "Bedrock models + flexible"],
  ["Data Integration", "40+ built-in connectors", "Knowledge Bases + OpenSearch"],
];

const tableX = 0.42;
const tableY = 1.41;
const tableW = 12.5;
const headerH = 0.6;
const rowH = 0.68;
const colWidths = [3.0, 4.5, 4.5];  // Proportional column widths

// Normalize column widths to fit tableW
const totalColW = colWidths.reduce((a, b) => a + b, 0);
const colW = colWidths.map(w => (w / totalColW) * tableW);

// Header row
let xPos = tableX;
headers.forEach((header, c) => {
  slide.addShape(pres.shapes.RECTANGLE, {
    x: xPos, y: tableY, w: colW[c], h: headerH,
    fill: { color: AWS_THEME.colors.darkNavy }
  });
  slide.addText(header, {
    x: xPos, y: tableY, w: colW[c], h: headerH,
    fontSize: 15, fontFace: AWS_THEME.fonts.heading,
    color: c === 0 ? AWS_THEME.colors.slateGray : AWS_THEME.colors.white,
    bold: true, align: "center", valign: "middle", margin: [0, 8, 0, 8]
  });
  xPos += colW[c];
});

// Data rows (alternating fills)
rows.forEach((row, r) => {
  const y = tableY + headerH + r * rowH;
  const fillColor = r % 2 === 0 ? "0D1117" : "161E2D";

  xPos = tableX;
  row.forEach((cell, c) => {
    slide.addShape(pres.shapes.RECTANGLE, {
      x: xPos, y: y, w: colW[c], h: rowH,
      fill: { color: fillColor },
      line: { color: AWS_THEME.colors.slateGray, width: 0.3, transparency: 70 }
    });
    slide.addText(cell, {
      x: xPos, y: y, w: colW[c], h: rowH,
      fontSize: 15, fontFace: AWS_THEME.fonts.body,
      color: c === 0 ? AWS_THEME.colors.orange : AWS_THEME.colors.white,
      bold: c === 0, align: c === 0 ? "left" : "center", valign: "middle",
      margin: [0, 8, 0, 8]
    });
    xPos += colW[c];
  });
});

// Bottom recommendation bar
addSummaryBar(slide, "Recommendation: Option A for fast deployment, Option B for high customization, or hybrid approach");

addAwsFooter(slide);
```

## Full Image Background (sd35l)

Generated image as the entire slide background with a dark overlay for text readability.
Uses sd35l to create an atmospheric, abstract background.
Best for impactful quote slides, key message slides, and section transitions.

**Image generation:**
```bash
# Generate full background (16:9)
SD35L_SCRIPT=$(find ~/.claude/plugins -path "*/sd35l/scripts/generate_image.py" 2>/dev/null | head -1)
python3 "$SD35L_SCRIPT" \
  --prompt "Abstract dark cosmic nebula, deep space with subtle blue and purple gas clouds, scattered tiny stars, very dark overall tone, cinematic atmosphere, minimal clean composition, suitable as presentation background" \
  --negative-prompt "text, watermarks, bright colors, people, objects, busy details, white, light background" \
  --aspect-ratio 16:9 --seed 60001 \
  --output-dir /tmp/myslide-assets/
```

```javascript
let slide = pres.addSlide();
```

**Overlay guidelines:**
- Very dark images (space, deep navy): transparency 50-60 (lighter overlay)

- Add a subtle vignette effect by stacking overlays if needed

```javascript
function addSummaryBar(slide, text, options = {}) {
  const {
```
